# Козметично Студио - Система за резервации

Пълнофункционална система за управление на резервации за козметично студио с администраторски панел.

## Функции

### За клиентите
- 👤 Регистрация и вход
- 💅 Преглед на услуги с филтриране по категории
- 📅 Резервиране на часове
- ⏰ Преглед на собствените резервации
- 📝 Управление на резервациите

### За администраторите
- 📊 Контролен панел с статистика
- 🛠️ Управление на услуги (добавяне, редактиране, изтриване)
- 📋 Управление на резервации (потвърждение, отмяна)
- 👥 Преглед на потребителите
- ⚙️ Настройки на работното време

## Технологии

- **Frontend**: HTML5, CSS3, JavaScript (ES6+)
- **Backend**: Firebase (Authentication, Firestore)
- **Архитектура**: Component-based, State management

## Инсталация и стартиране

### 1. Подготовка

Проверете че имате `node.js` и `npm` инсталирани.

### 2. Конфигуриране на Firebase

Firebase конфигурацията е вече зададена в `js/config/firebase.js` с публичния API ключ на студиото.

### 3. Стартиране

За локално развитие, трябва да използвате локален сървър:

```bash
# Използвайте Python 3
python -m http.server 8000

# Или Node.js с http-server
npx http-server -p 8000
```

След това посетете `http://localhost:8000`

## Начало на работа

### За клиентите

1. **Регистрация**
   - Кликнете "Регистрация" в горната част
   - Попълнете име, телефон, имейл и парола
   - Вече сте регистрирани и можете да направите резервация

2. **Резервирайте услуга**
   - Изберете услуга от списъка
   - Попълнете дата и час
   - Потвърдете резервацията

3. **Управление на резервациите**
   - Влезте в профила си
   - Преглед на предстоящите и минали резервации
   - Отмяна на резервация ако е необходимо

### За администраторите

1. **Регистрирайте се като администратор (първи път)**
   - Регистрирайте нов акаунт с произволен имейл и парола
   - Чрез Firebase Console: 
     - Отворете https://console.firebase.google.com
     - Отидете във вашия проект > Firestore Database > users колекция
     - Намерете потребителя и добавете поле `role: "admin"`

2. **Вход като администратор**
   - Кликнете "Администрация" в навигацията (видимо само за администратори)
   - Или посетете директно `/admin.html`

3. **Зареждане на демо данни**
   - Приложението вече не добавя примерни данни автоматично от браузъра.
   - За пълен reset на Firestore и презапис с нови демо данни изпълнете:
     ```bash
     npm run demo:reset
     ```
   - Командата изчиства колекциите `bookings`, `services`, `users`, `schedule`, `settings` и създава нови демо акаунти, услуги, настройки и резервации директно във Firebase.

4. **Контролен панел**
   - Преглед на статистика и графики
   - Управление на услугите
   - Управление на резервациите
   - Преглед на потребителите
   - Конфигуриране на настройки

## API услуги

### AuthService
- `register(email, password, displayName, phone)` - Регистрация
- `login(email, password)` - Вход
- `logout()` - Изход
- `resetPassword(email)` - Възстановяване на парола

### BookingService
- `getAvailableSlots(date, duration)` - Получаване на свободни часове
- `createBooking(bookingData)` - Създаване на резервация
- `cancelBooking(bookingId)` - Отмяна на резервация
- `getUserBookings(userId)` - Получаване на потребителски резервации
- `getAllBookings(filters)` - Получаване на всички резервации (администратор)
- `getBookingStats(startDate, endDate)` - Получаване на статистика

### DataService
- `getServices()` - Получаване на услуги
- `addService(serviceData)` - Добавяне на услуга
- `updateService(id, updates)` - Редактиране на услуга
- `deleteService(id)` - Изтриване на услуга
- `getBookings(filters)` - Получаване на резервации

## Функции за администраторите

### Управление на услуги
- Просмотр на всички услуги
- Добавяне на нови услуги
- Редактиране на съществуващи услуги
- Изтриване на услуги
- Сортиране по ред на показване

### Управление на резервации
- Преглед на всички резервации
- Потвърждение на резервации
- Отмяна на резервации
- Филтрирање по дата и статус
- Преглед на детайли за резервация

### Потребителски профили
- Списък на всички потребители
- Статистика за всеки потребител (брой резервации, похарчени суми)
- Най-активни клиенти

### Настройки
- Конфигуриране на работното време (начало/край)
- Конфигуриране на продължителност на процедури
- Експортиране на данни
- Изтриване на минали резервации

## Валидиране

### Форми
- Email валидиране
- Телефонен номер валидиране (български формат)
- Парола валидиране (минимум 6 символа)
- Име валидиране
- Дата валидиране
- Дължина на текст валидиране

### Резервации
- Дата проверка (не в миналото, не повече от 3 месеца напред)
- Час проверка (работно време)
- Проверка за конфликти с други резервации
